Description

List your notifications. List all notifications for the current user, grouped by repository.

URI Parameters

mediaTypeExtension required, one of (.json)

Use .json to specify application/json media type.

Example: .json

Headers

Accept string

Is used to set specified media type.

X-GitHub-Media-Type string

You can check the current version of media type in responses.

X-GitHub-Request-Id integer

X-RateLimit-Limit integer

X-RateLimit-Remaining integer

X-RateLimit-Reset integer

Query Parameters

all string

True to show notifications marked as read.

participating string

True to show only notifications in which the user is directly participating or mentioned.

since string

Time filters out any notifications updated before the given time. The time should be passed in as UTC in the ISO 8601 format: YYYY-MM-DDTHH:MM:SSZ. Example: "2012-10-09T23:39:01Z".

max_id integer

Returns results with an ID less than (that is, older than) or equal to the specified ID.

since_id integer

Returns results with an ID greater than (that is, more recent than) the specified ID. There are limits to the number of Tweets which can be accessed through the API. If the limit of Tweets has occured since the sinceid, the sinceid will be forced to the oldest ID available.

trim_user one of (0, 1, true, false, t, f)

When set to either true, t or 1, each tweet returned in a timeline will include a user object including only the status authors numerical ID. Omit this parameter to receive the complete user object.
